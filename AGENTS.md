# AGENTS.md — effect-starter

## Monorepo structure

pnpm workspace with three packages:

- `apps/api` — Effect Platform HTTP server, Drizzle ORM + PostgreSQL
- `apps/web` — React SPA (TanStack Router, TanStack Query, Tailwind v4, shadcn/ui New York)
- `packages/contracts` — shared Effect Schema types and HTTP API definitions


## Dev commands (run from root)

| Command | What |
|---|---|
| `pnpm lint` | oxlint (not ESLint) |
| `pnpm lint:fix` | oxlint --fix |
| `pnpm format` | oxfmt (not Prettier) |
| `pnpm format:check` | oxfmt --check |
| `pnpm typecheck` | tsc --noEmit in every package |
| `pnpm test` | vitest run in every package |
| `pnpm build` | tsup for api, vite build for web |
| `pnpm -r --parallel <script>` | run a script in all packages |


## Code conventions

- **`verbatimModuleSyntax`** — use `import type` for type-only imports.
- **`typescript/consistent-type-imports`** enforced by oxlint.
- **`noUncheckedIndexedAccess`** — array[index] returns `T | undefined`.
- **`.js` extensions** in all local import paths (ESM).
- **Effect Platform** — contracts use `HttpApi`/`HttpApiGroup`/`HttpApiEndpoint`; handlers use `HttpApiBuilder.group()`.
- **Drizzle** schema lives in `apps/api/src/db/schema.ts`. Drizzle config reads `DATABASE_URL`.

## Generated / ignored files

| File | Purpose | Notes |
|---|---|---|
| `apps/web/src/routeTree.gen.ts` | TanStack Router codegen | Auto-regenerated by Vite plugin on dev/build. Ignored by oxlint and oxfmt. |
| `apps/web/.tanstack/` | TanStack Router cache | Do not touch manually. |
| `apps/api/drizzle/` | Drizzle Kit migrations | Generate with `pnpm --filter @effect-starter/api db:generate`. |

## Testing

- **Vitest** in every package. Run focused with `pnpm --filter <pkg> test`.
- Contracts package has `vitest run` (no `--passWithNoTests` flag, will fail if no test files found).
- API and web packages use `vitest run --passWithNoTests`.

## Key package exports

`@effect-starter/contracts` has explicit subpath exports:
- `@effect-starter/contracts/http`
- `@effect-starter/contracts/schemas/user`
